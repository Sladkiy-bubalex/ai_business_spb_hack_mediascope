# Mediascope — AI Business SPB Hackathon 2026

Классифицируйте поисковые запросы по трём атрибутам:

- **TypeQuery** (бинарный) — относится ли запрос к профессиональному видеоконтенту
- **ContentType** — `фильм`, `сериал`, `мультфильм`, `мультсериал`, `прочее` или пусто
- **Title** — нормализованное название франшизы или пусто

Тренировочные данные содержат пользовательские запросы с опечатками, транслитерацией и обобщёнными формулировками. Тестовые данные командам недоступны — вы отправляете код, который запускается в серверной песочнице.

**Платформа:** https://app.ai-business-spb.ru

## Описание задачи

**АО «Медиаскоп»** — лидер российского рынка медиаисследований, единый измеритель аудитории ТВ и интернета по решению Роскомнадзора. Более 30 лет компания измеряет аудиторию всех медиа в России. Данные Mediascope — основа для принятия решений крупнейших телеканалов, онлайн-площадок, рекламных агентств и рекламодателей.

В рамках проекта по исследованию интереса к профессиональному видеоконтенту Mediascope обрабатывает поток поисковых запросов пользователей — тысячи разнообразных формулировок вроде `«1 сезон тьмы»`, `«смотреть фильмы 2025 года онлайн»` или `«10 троллейбус ижевск»`.

Сегодня разметка этих запросов выполняется операторами вручную: такой подход обеспечивает высокую точность, но требует значительных временны́х и человеческих ресурсов.

**Роль ИИ** — упростить работу операторов: уйти от разметки «с нуля» и перейти к проверке уже размеченных моделью запросов. Возможна схема, при которой запросы с высокой уверенностью модели не проверяются операторами вовсе, а с низкой — передаются на ручную проверку.

## Цель и ожидаемый результат

Необходимо разработать **ML-модель**, которая принимает на вход сырые формулировки поисковых запросов и возвращает их с разметкой по трём признакам:

| # | Признак | Описание |
|---|---------|----------|
| 1 | **Принадлежность к видеоконтенту** | Относится / не относится к профессиональному видеоконтенту |
| 2 | **Тип контента** | `сериал` / `фильм` / `мультфильм` / `мультсериал` / `null` |
| 3 | **Название контента** | Название произведения или франшизы / `null` |

**Примечание по франшизам:** если запрашиваемый контент относится к франшизе, в качестве названия выводится название франшизы. Пример: для запроса `«гарри поттер и узник азкабана смотреть в full hd»` — название: `гарри поттер`.

## Бизнес-ценность и потенциал внедрения

Модель будет встроена в пайплайн обработки данных в виде трансформации в **DAG Airflow**:

* **Вход:** датасет с сырыми формулировками запросов
* **Выход:** те же формулировки + три заполненных столбца с параметрами разметки

Успешное внедрение позволит:

* сократить затраты операторов на ручную разметку;
* увеличить объём обрабатываемых данных;
* расширить выборку респондентов и повысить качество рассчитываемых показателей исследования.

**Режим данных:** коммерческая тайна — конфиденциальная бизнес-информация. **Требования к развёртыванию:** on-premise, open-source решения.
**Время обработки:** Обработка 15 тыс. запросов не должна превышать 4 часа.

## Основные функции системы

* **Бинарная классификация** запросов по признаку интенции к просмотру профессионального видеоконтента (`1` — относится, `0` — не относится).
* **Классификация по типу контента** на 4 категории: `фильм`, `сериал`, `мультфильм`, `мультсериал`.
* **Извлечение названия контента**, включая определение принадлежности к франшизе.
* **Оценка уверенности** модели в предсказании — для маршрутизации запросов между автоматической разметкой и ручной проверкой оператором.
* **Поддержка работы с новым контентом** — обработка запросов по произведениям, вышедшим после последней даты обучения.
* **Переобучение модели** на новых размеченных данных — с учётом ежемесячного обновления данных и непрерывного выхода нового контента.

## Особенности задачи

* **Новый контент:** модель должна корректно обрабатывать запросы по произведениям, вышедшим после последней даты обучения. Полностью предугадать появление нового контента невозможно — это ключевая сложность задачи.
* **Регулярное переобучение:** данные обновляются ежемесячно; прототип должен поддерживать инкрементальное или периодическое переобучение на новых размеченных данных.
* **Размеченные данные:** предоставляются и будут пополняться по мере работы.

## Метрика

```
typequery_f2         = F_beta(y_true, y_pred, beta=2)                   # по всем строкам
contenttype_macro_f1 = macro F1 по 6 классам                            # только GT TypeQuery=1
title_token_f1       = среднее token-level F1 на bag-of-words           # только GT TypeQuery=1
combined_score       = 0.35 * typequery_f2 + 0.30 * contenttype_macro_f1 + 0.35 * title_token_f1
```

`combined_score` — ключ лидерборда.
